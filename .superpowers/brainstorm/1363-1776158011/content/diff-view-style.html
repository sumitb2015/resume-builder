What should the Session Diff view look like?

The existing "Compare Resumes" is a side-by-side rendered resume view. Here are 3 styles for the new session diff. Pick what feels most useful.

A

Reuse existing side-by-side rendered resume view

Same as current "Compare Resumes" modal — original left, current right. New/changed bullets highlighted green.

Pros

  • Zero new code — reuses ResumeComparisonModal
  • Familiar UX for users who've used Compare
  • Renders actual formatted resume

Cons

  • Hard to spot small text changes in rendered view
  • No change summary / count
B

Change list — "what changed" summary panel

Shows each changed field as a before/after pair with section label. Like a PR diff, but for resume content.

Pros

  • Easiest to scan — see exact text changes
  • Change count badge shows impact at a glance
  • Git-like UX users already understand

Cons

  • Requires diffing logic between two Resume objects
  • No rendered resume preview
C

Hybrid — change list on left + rendered diff on right

Left panel lists all changes; right panel shows full side-by-side rendered view. Best of both, larger modal.
